**a) Сигурен ли си, че въведе токена, а не GitHub паролата?**

**b) Сигурен ли си, че токенът има `repo` scope?**
Виж тук: https://github.com/settings/tokens — кликни на токена и провери чекбокса "repo".

**c) Ако използваше fine-grained token:**
Fine-grained токените не работят, ако не си дал експлицитен достъп до репото `amutafis/banking-system`. По-сигурно е да си направиш **classic** token: https://github.com/settings/tokens/new (не /beta).

## Може ли без Makefile? Да.

Какво прави Makefile-ът:
- Автоматизира компилацията — с `make` вместо ръчно `g++` всеки път
- Тракирa dependencies — пре-компилира само променените файлове (бързо при нарастване на проекта)
- Чисти с `make clean`

Без Makefile, една команда върши работа:
```sh
g++ -std=c++17 -Iinclude src/*.cpp -o banking
./banking
```

Или като shell скрипт `build.sh`:
```sh
#!/bin/bash
g++ -std=c++17 -Iinclude src/*.cpp -o banking
```

**За училищен проект:** Makefile-ът е стандартна практика и хубаво е да си го оставиш, но НЕ е задължителен и за защита трябва да можеш да обясниш всеки ред в него (`$@`, `$^`, `%.o: %.cpp` са специфичен синтаксис). Ако ти изглежда сложен — просто го махни и използвай ръчна команда.

**Какво искаш?**
- (a) Оставяме Makefile-а, обяснявам ти всеки ред в него
- (b) Махаме го, документирам ръчната `g++` команда в README

Ако кажеш кое, ще го направя заедно с README-то. Засега ще пиша README с **двата варианта** — с `make` и с директната `g++` команда — за да си гъвкав.
